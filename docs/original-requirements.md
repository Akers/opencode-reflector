# 任务简介及需求
开发一个中文名叫反思怪、英文名opencode-reflector，为opencode打造的反思智能体，通过从session中分析并记录任务完成情况、任务完成耗时、参与任务的智能体、参与任务的智能体消耗tokens和请求次数、任务消耗的总tokens和请求次数、人类参与时长、人类参与度、使用工具/MPC/Skills清单、各工具/MPC/Skills使用次数等监控指标、人类情绪分析及趋势，为智能体/智能体的框架选型、自我进化提供基础数据支撑的工具。

## 兼容的智能体工具
- opencode: 插件开发参考文档https://opencode.ai/docs/plugins/
- openclaw: 插件开发参考文档https://docs.openclaw.ai/plugins/building-plugins
- claudecode: 插件开发参考文档https://code.claude.com/docs/en/plugins

## 需求简要说明
### 工具配置
#### 触发方式
触发方式分为：
1. 定时触发/TIME，每天定时触发，默认每天0点触发
2. 事件触发/EVENTS，支持用户选择以下触发事件，并支持配置多个：
2.1. 任务完成/TASK_FINISHED: 如用户使用Openspec，则用户发送/opsx:archive指令且智能体完成该指令后执行反思；如用户未使用/opsx:archive指令，则需要获取会话中最后一个消息为Agent消息，且其中包含完成工作、任务已完成等和任务结束的相关表述则视为任务完成。
2.2. 10轮对话/10_MESSAGES，每完成10轮对话时触发
3. 手动触发/MANUAL：当用户输入/or:reflect_now的时候触发

#### 触发时间
当触发方式为定时触发时有效，配置24小时制的时间，每天的该时间点触发反思

#### 模型配置
配置reflector使用的模型id，如"minimax-cn-coding-plan/MiniMax-M2.7-highspeed"，以当前Agent框架的API调用模型

### 工作流程
当reflector触发时，按照以下流程执行
获取会话Session --> 监控指标提取 --> 识别任务状态 --> 人类参与度分析 --> 数据持久化 --> 生成日报

#### 获取会话Session
获取当前智能体工具的截止上次本工具完成工作时间到当前时间的所有会话

#### 监控指标提取
以Session为主体，记录每个session id的监控指标

#### 识别任务状态
读取最后一轮对话内容，使用关键字识别判断任务是否成功

#### 人类参与度分析
按时序读取对话中的人类发送的消息进行分析，人类认可度、人类负面情绪占比、人类参与工作类型、人类参与时长、人类介入次数

#### 生成日报
在当前项目的.reflector目录中，以dayreport-<report-day>.md为文件名生成日报，其中如反思执行时间是00:00到12:00之间，则report-day为当前日期-1天，如果执行时间是12:00到24:00期间，则report-day为当前日期。日报包含以下内容：
1. 当日任务清单，每个会话视为一个任务，需要展示：会话id，会话标题、任务开始时间、任务结束时间、会话Token消耗、人类参与度、人类负面情绪占比、任务完成情况。
2. 当日任务消耗的总tokens和请求次数、人类参与时长、人类参与度、人类态度占比的汇总统计
3. 当日使用工具/MPC/Skills清单、各工具/MPC/Skills使用次数汇总统计

### 数据指标
以下为本工具需要采集的数据指标
- 任务开始时间：当前会话第一个消息的产生时间
- 任务结束时间：当前会话的最后一个消息产生时间
- 任务完成情况：完成（最后一个消息是智能体回复且包含任务完成的相关信息）、已中断（最后一个消息是智能体回复且未包含任务完成的相关信息）
- 会话Token消耗：当前会话调用大模型的Token消耗，包含prompt token 和 completion token
- 会话模型请求次数：当前会话调用大模型的请求次数，一次上行一次下行为以次
- 会话工具调用次数：当前会话的工具调用次数，并以工具名称分组统计调用次数，如read: 7,write: 100
- 会话工具调用清单：以会话id为索引，记录会话调用的工具清单，需保存工具名、工具调用耗时、工具调用是否成功
- 会话MCP调用次数：当前会话的MCP调用次数，并以工具名称分组统计调用次数，如contex7: 7
- 会话MCP调用清单：以会话id为索引，记录会话调用的MCP清单，需保存MCP名、MCP调用耗时、MCP调用是否成功
- 会话SKILLS调用次数：当前会话的SKILLS调用次数，并以SKILLS名称分组统计调用次数，如pptx: 1, pd: 10
- 会话SKILLS调用清单：以会话id为索引，记录会话调用的SKILLS清单，需保存SKILLS名
- 会话智能体/子代理数量：参与当前会话的智能体及子代理数量
- 会话中消息数：当前会话的消息数，每次工具、MCP调用都算一个消息
- 会话总智能体消息数：当前会话提出人类参与的消息外的消息总数
- 会话智能体/子代理参与度：统计当前会话中，每个智能体/子代理的消息数和总智能体消息数的百分比
- 会话人类参与次数：统计会话中人类发送指令/消息的数量
- 会话人类参与率：统计会话中人类发送指令/消息的数量和总消息数的百分比
- 会话中断次数：统计会话中的任务中断数量，以及包含“任务中断”、“恢复工作”、“继续完成.....工作”、“继续执行”等和工作中断及恢复相关的消息的数量
- 会话人类负面情绪占比：分析会话中所有人类消息，通过情感分析统计包含负面情绪消息数量的占比
- 会话人类态度评分：分析会话中所有人类消息，通过情感分析，对当前会话中人类的情绪进行打分，按1到10打分，1为极端暴躁、质疑、反感情绪，10为对智能体工作非常满意
- 会话人类认可度：分析会话中的所有人类消息，判断人类对于Agent工作成功的认可程度，按1到10分打分

# 架构设计规范
## 技术选型要求
- 当需要对智能体工具进行扩展时，使用官方指定的方式进行插件扩展，不考虑修改其源码
- 涉及本地文件读写时，使用go语言编写高性能的文件读写工具
- 涉及创建智能体时，智能体的所有提示词都应使用.md文本文档进行存储，且支持热修改
- 工具应以hook方式提供便捷的自定义扩展，支持的hook节点：保存指标前、保存指标后、情感分析前、情感分析后
